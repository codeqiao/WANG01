# 需求

## 导航条
```
    1.通栏到头，固定到顶部
    2.点击登录链接弹出居中登录对话框
    3.样式和间距按照效果制作
```

## 登录窗口
```
    1.居中显示
    2.带圆角和阴影
    3.点击login按钮后自动验证用户名和密码是否为空
    4.如果不为空，则提交，否则，出现错误提示
```

## 列表页面
```
    1.在导航栏中点击进入
    2.有列表标题
    3.图文并茂的列表内容在左侧
    4.在右侧有一个特别推荐的图片列表
    5.
```





# 项目开发流程

```
    1.需求>需求文档
        1.功能
        2.文档
        3.流程
        4.时间
    2.技术分析>针对文档来分析
        1.使用什么技术
        2.人员分工
        3.每个人完成的周期
    3.项目实际开发>针对技术和人员各自开发
        1.每周站会-总结技术中碰到的问题
        2.每天有10分钟的例会-知道昨天作了什么，今天要做什么。
    4.项目测试>由测试人员根据需求完成测试工作
        1.功能测试
        2.BUG提交--修改BUG
        3.编写测试报告
    5.项目的部署>源码的提交
        1.在正式服务器上部署
        2.测试人员在正式服务器上回测
        3.完成项目上线
```
